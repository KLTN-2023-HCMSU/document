Thuật toán

Chọn hướng tiếp cận: rule-based trước, ML sau

Hướng Điểm mạnh Giới hạn Vai trò trong đề tài
Rule-based Giải thích rõ, triển khai nhanh, dễ kiểm thử, không cần dữ liệu lớn. Cần cập nhật rule thủ công, có thể bị học pattern. MVP chính: scoring 0-100, SAFE/CAUTION/DANGER, trả evidence.
Machine Learning Tự học pattern, phát hiện mẫu mới, độ chính xác nghiên cứu thường cao. Cần dataset đã label, chi phí training, khó giải thích hơn. Hướng mở rộng: TF-IDF/Naive Bayes cho text, Random Forest/XGBoost cho URL.
Hybrid Rule xử lý nhanh case rõ ràng, ML xử lý vùng không chắc chắn. Phức tạp hơn vì phải duy trì hai lớp phân tích. Đích dài hạn: vẫn giữ giải thích được, nhưng tăng khả năng bắt mẫu mới.

Khuyến nghị trong khóa luận: bắt đầu bằng Rule Engine + Heuristic Analysis, sau đó trình bày ML như lộ trình phát triển.

URL + Form

Phân tích link và trang nhập liệu bằng tín hiệu có thể giải thích

URL Scanner

Feature extraction

  • Chuẩn hóa URL, decode encoding, expand shortener nếu có.
  • Kiểm tra độ dài, subdomain, IP thay domain, HTTPS, ký tự @ hoặc //.
  • Domain age, redirect chain, entropy và tỷ lệ ký tự đặc biệt là tín hiệu mở rộng.
Reputation

Blacklist + similarity

  • Domain blacklist cộng điểm mạnh; whitelist trừ điểm.
  • Redirect chain mỗi bước có thể tăng rủi ro.
  • Typosquatting dùng Levenshtein hoặc Jaro-Winkler so với brand phổ biến tại Việt Nam.
Form/HTML

Form đáng ngờ

  • Parse bằng Jsoup, trích xuất form và input.
  • Cộng điểm khi có OTP, PIN, CVV, CCCD, tài khoản ngân hàng.
  • Cảnh báo form gửi sang domain khác, không HTTPS hoặc dùng từ khóa dụ dỗ.

Text analyzer

Tin nhắn được xử lý bằng entity, keyword và pattern kết hợp

Tiền xử lý

Lowercase, chuẩn hóa số điện thoại/số tiền, loại ký tự nhiễu và tách câu đơn giản; không cần NLP phức tạp cho MVP.

Entity extraction

Regex trích URL, số điện thoại Việt Nam, số tài khoản, số tiền, OTP và deadline như "trong X phút".

Keyword dictionary

Nhóm từ khẩn cấp, giả mạo cơ quan, yêu cầu thông tin nhạy cảm, chuyển tiền và "quá tốt để tin".

Composite rules

Nhận diện SMS giả mạo ngân hàng, tuyển dụng thu phí, shipper giả, hoàn tiền giả, hoặc khẩn cấp kèm yêu cầu OTP.

Cross-reference

Nếu tin nhắn chứa URL, số điện thoại hoặc STK, hệ thống gọi tiếp URL Scanner và Phone/Bank Checker.

Output

Tổng điểm được cắt về 100, phân loại SAFE/CAUTION/DANGER và giải thích bằng các evidence cụ thể.

Phone + QR

Số điện thoại, tài khoản ngân hàng và QR dùng lookup cộng đồng

Phone/Bank Checker
  • Chuẩn hóa input: phone về canonical form, STK bỏ space/dash.
  • Hash SHA-256 để hạn chế lưu dữ liệu nhạy cảm dạng thô.
  • Tra blacklist/whitelist, số báo cáo, thời gian báo cáo gần nhất và uy tín người báo cáo.
  • Time-decay: báo cáo mới có trọng số cao hơn báo cáo cũ.
QR Scanner
  • Decode bằng ZXing để lấy raw data.
  • Phân loại URL, VietQR/banking format, plain text hoặc unknown.
  • VietQR được parse bank_id, account_number, amount và content.
  • QR tiếp tục được định tuyến sang URL/Text/Bank Checker để tính điểm tổng hợp.

Rule engine

Core engine cần tách rule, condition và evidence

Rule Repository

  • Lưu RiskRule, RuleCondition, RuleAction.
  • Rule có code, name, weight, enabled.
  • Có DB + cache để cập nhật rule mà không sửa code lõi.

Rule Evaluator

  • Duyệt active rules và kiểm tra matches(input).
  • Cộng weight vào tổng điểm.
  • Thu thập evidence gồm mã rule, tên rule, điểm và giải thích.

Rule Composer

  • Hỗ trợ AND / OR / NOT cho composite rules.
  • Ưu tiên, weight và threshold mapping.
  • Phân loại: dưới 30 SAFE, 30-59 CAUTION, từ 60 DANGER.

ML roadmap

ML chỉ nên thêm khi có dữ liệu và baseline rõ

Điều kiện chuyển sang ML
  • Có dataset từ 10.000 mẫu đã label trở lên.
  • Rule-based đã có baseline nhưng false positive còn nhiều.
  • Team đủ năng lực ML hoặc có nguồn lực vận hành model.
  • Cần phát hiện pattern mới chưa biết trước.
Roadmap
  • Thu thập PhishTank, Alexa/legitimate URLs, Tín Nhiệm Mạng, user reports.
  • Feature engineering với URL, domain, content và similarity features.
  • Ưu tiên Random Forest/XGBoost vì có feature importance.
  • KPI: precision >= 95%, recall >= 90%, inference dưới 100ms.

Hybrid dài hạn: rule engine xử lý nhanh case rõ ràng; chỉ đẩy vùng 20-70 điểm sang ML classifier.

Performance

Thiết kế phải đủ nhanh cho trải nghiệm scan tức thời

Cache

  • L1 Caffeine cho domain reputation, phone/account lookup, recent scan.
  • L2 Redis cho blacklist/whitelist, prediction và thống kê tổng hợp.
  • TTL theo loại dữ liệu: 5 phút, 15 phút, 1 giờ hoặc 24 giờ.

Database

  • Index theo entity_type + value_hash.
  • Index status cho verified entities.
  • Index lịch sử scan theo user và thời gian.
  • Audit log có thể partition theo tháng.

Async

  • Phân tích URL có thể chạy background.
  • Domain, SSL và content checks chạy song song.
  • Kết quả được combine trước khi trả risk score.

Quyền mobile

Không vượt quyền là quyết định thiết kế

Vì sao không vượt quyền
  • Pháp lý: dữ liệu SMS, call log, notification và clipboard đều nhạy cảm.
  • Kỹ thuật: Android/iOS chặn hoặc giới hạn mạnh việc đọc nền.
  • Google Play policy hạn chế quyền SMS/Call Log, Accessibility và background access.
  • OWASP Mobile khuyến nghị permission tối thiểu, consent rõ ràng và data minimization.
Quyền MVP hợp lý
  • Internet/network state cho gọi API backend.
  • Camera runtime permission chỉ khi user quét QR.
  • Share Target để nhận link/text do user chủ động chia sẻ.
  • Notification của chính app để trả kết quả hoặc nhắc cảnh báo, không đọc app khác.

Thông điệp khi thuyết trình: bảo vệ người dùng phải đi cùng privacy, consent và chính sách nền tảng.

Mobile flows

Thay "chạy ngầm" bằng luồng user chủ động

Share Target

User share link từ SMS, Zalo, browser hoặc app khác sang Anti-Scam để quét.

QR Scanner

User mở app, bật camera runtime, scan QR và xem cảnh báo trước khi chuyển tiền.

Manual Input

Nhập URL, số điện thoại, STK hoặc nội dung tin nhắn khi nền tảng không cho đọc tự động.

Clipboard foreground

Chỉ đọc clipboard khi app đang mở và user bấm kiểm tra nội dung vừa copy.

API backend

API bao phủ auth, scan, history, report và admin

Nhóm API Endpoint chính Vai trò
Authentication /auth/register, /auth/login, /auth/refresh JWT access/refresh token, phân quyền user/admin.
Scan /scan/url, /scan/text, /scan/phone, /scan/bank-account, /scan/qr Nhận input từ app, chạy module phân tích và trả risk score.
History & Reports /scan/history, /scan/{scanId}, /reports, /reports/my Lưu lịch sử scan, xem chi tiết và gửi báo cáo lừa đảo.
Admin Rules, risk entities, report review, dashboard, trend analysis. Quản trị rule, blacklist/whitelist, kiểm duyệt report và theo dõi hệ thống.

API response

Kết quả API phải phục vụ quyết định của user

Response scan

  • riskScore: 0-100.
  • riskLevel: SAFE, CAUTION hoặc DANGER.
  • analysis: features, entities và cross-reference.
  • evidences: rule match, severity, message.
  • recommendations: BLOCK, VERIFY và next steps.

Chuẩn vận hành

  • Success/error response thống nhất.
  • Error codes cho validation, auth, rate limit và scan failure.
  • Rate limiting theo nhóm endpoint để chống abuse.
  • Webhook optional cho tích hợp tương lai.
