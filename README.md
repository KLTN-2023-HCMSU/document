# Quản lý Tài liệu Khóa luận (BA & Project Docs)

Thư mục này được sử dụng để lưu trữ toàn bộ tài liệu liên quan đến việc phân tích nghiệp vụ (BA), khảo sát thị trường, kế hoạch dự án và các tài liệu tham khảo cho hệ thống Anti-Phishing Scam Warning.

Để đảm bảo tài liệu luôn dễ tìm kiếm và logic, team vui lòng tuân thủ cấu trúc thư mục dưới đây:

## 📂 Cấu trúc thư mục

* **`00_Quy_Trinh/`**
  Nơi lưu trữ các biểu mẫu (template) tài liệu, quy trình làm việc chung của team (ví dụ: cách viết báo cáo, cách đặt tên file).

* **`01_De_Xuat_Du_An/`**
  Lưu các tài liệu liên quan đến ý tưởng, đề tài khóa luận ban đầu và các khía cạnh kỹ thuật tổng quan.

* **`02_Phan_Tich_Nghiep_Vu/`**
  Lưu các tài liệu khảo sát, phân tích thị trường, tài liệu đặc tả yêu cầu (SRS, Use Case, v.v.).

* **`03_Ke_Hoach_Va_Bao_Cao/`**
  Lưu các file về planning hàng tuần, báo cáo tiến độ (Weekly Report).

* **`04_Hop_Va_Danh_Gia/`**
  Lưu trữ feedback từ giảng viên, biên bản họp team.

* **`05_Tai_Lieu_Tham_Khao/`**
  Lưu các tài liệu tham khảo bên ngoài như tiêu chuẩn OWASP, tài liệu nghiên cứu.

* **`06_Thiet_Ke/`**
  Dành cho việc lưu trữ các thiết kế UI/UX, Diagram (Sequence, Class, v.v.) sau này.

## 📝 Quy tắc đặt tên file

1. **Không sử dụng khoảng trắng**: Dùng dấu gạch dưới `_` hoặc gạch nối `-` (VD: `Khao_sat_thi_truong.md` thay vì `Khao sat thi truong.md`).
2. **Tiếng Việt không dấu**: Khuyến khích dùng tiếng Việt không dấu để tránh lỗi font khi mở trên các hệ điều hành khác nhau.
3. **Thêm ngày tháng (nếu cần)**: Với các báo cáo tiến độ, nên thêm ngày tháng vào cuối tên file để dễ phân biệt (VD: `Weekly_Report_2026_07_14.md`).
